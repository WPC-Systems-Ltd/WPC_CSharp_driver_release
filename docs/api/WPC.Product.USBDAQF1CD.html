CAN_read(Int32, Int32, Int32)

Read frames from a given port.
After CAN communication starts, data received by the DAQ are streamed to the PC and stocked in a queue in the WPC device handle.
Users should call this function to retrieve them from the queue.

Declaration
public List<CANFrame> CAN_read(int port, int num_frames, int delay = 5)
Parameters
Type Name Description
System.Int32 port

See here for available ports of different WPC devices.
System.Int32 num_frames

Number of CAN frames to get out from the queue.
System.Int32 delay

Time in milliseconds to wait for the indicated amount of frames to arrive.
This becomes ineffective if there are already num_frames frames in the queue.
Returns
Type Description
List<CANFrame>

List of CANFrame object
List of received CAN frames.
For each element frame in frame_list,
frame.id gives the identifier of the frame;
frame.extended_id tells whether its format is extended;
frame.remote tells whether it is a remote frame;
frame.data gives the data.
